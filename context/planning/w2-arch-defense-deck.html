01 · The scenario

The signal is buried in the documents

A physician is prepping a follow-up visit. The chart has structured OpenEMR data — but the recent, important information sits in a scanned lab PDF and a front-desk intake form.

"What changed, what should I pay attention to, and what evidence supports the recommendation?"
  • See documents — ingest a lab PDF and an intake form, extract structured facts.
  • Separate fact from evidence — patient-record facts vs. guideline evidence, each cited.
  • Stay grounded when it's messy — imperfect scan, incomplete record, follow-up questions.

05 · Decision 1 — Agent framework Decided · Pydantic AI

Pydantic AI over LangGraph — concretely

LangGraph is a reasonable multi-agent default and owns one thing outright: a rendered graph. But our flow is shallow, and our verification gate is already first-class in Pydantic AI.

CriterionPydantic AILangGraph
Our flow
supervisor → 2 workers → verify		Delegation + hand-off express it directlyA full StateGraph for a shallow, near-linear flow
Verification gate@output_validator + ModelRetry — one hook, self-correctingRebuild it: a validate node + conditional edge + retry counter in graph state
Migration from today's agentNear-zero — tools, deps, contracts, Langfuse carry overRewrite the turn loop; port every tool to a node
Inspectable routing (PRD)Structured route events + Langfuse child spansNative graph object — its real edge
The gate decides it. LangGraph makes us rebuild self-correcting verification that Pydantic AI gives as one decorator — and PRD Core Req 4 accepts "another inspectable orchestration framework," i.e. inspectable handoffs, not a rendered graph.

OpenAI Agents SDK also considered — rejected: Claude second-class via LiteLLM, and its guardrails halt rather than self-correct.

JOS-45 When LangGraph wins: durable resumable state or human-in-the-loop mid-flow (§6.1 tripwires #1/#2 — not present today). In-framework escalation is pydantic-graph first, before any cross-framework move.

07 · Decision 3 — Hybrid RAG + rerank Decided · Qdrant

Ground the answer in guideline evidence

Qdrant (dedicated Railway service) · hybrid in one Universal Query API call · Cohere Rerank on the fused top-k · only top grounded snippets reach the answer model.

Qdrant ✓ LanceDB — runner-up pgvector — rejected (MySQL, not Postgres) Weaviate · Chroma
  • Native sparse + dense + RRF fusion in one API call — rank-based fusion sidesteps the BM25-vs-cosine score-scale problem (no alpha to tune or defend).
  • One added service, not two — FastEmbed folds embedding + sparse encoding into the client; yields a real /ready vector-index dependency an in-process store can't.
  • Reranker — Cohere rerank-v4.0-fast, the PRD-named default (~$2/1k, negligible on a small corpus).
JOS-46 Switch to LanceDB (embedded) if a second always-on service isn't worth it for a few-hundred-chunk static corpus.

09 · Decision 5 — Data authority & lineage Partial

One source of truth per data type

Four new data types enter the system. Each needs an owner, lineage, access control, and validation — no silent overwrites.

  • Extracted lab observations — derived → FHIR Observation, round-tripped.
  • Intake facts — demographics, meds, allergies, family history.
  • Guideline chunks — corpus, versioned, reproducible from repo.
  • Citation records — link every claim back to a source.
Uploaded documents and derived observations must round-trip through OpenEMR without creating duplicate or untraceable records.
→ Task Data-model + lineage + access-control spec.

12 · Risks & tradeoffs

Where this fails, and the mitigation

VLM invents field labels

Schema + citation + confidence make unsupported extracted facts visible, not silent.

Supervisor becomes a black box

Every routing decision and handoff is logged and explainable.

Scope creep — 5 doc types

Ship two reliably before adding a third. Narrower is stronger.

Multi-agent latency vs. <15s

Routing discipline + streaming; tiered models on cheap sub-tasks.

PHI leakage into SaaS observability is the disqualifier — scrubbing + a CI PHI-detection check guard it.
