2 · Panel loading state

Between submit and response the round-trip is a grounded FHIR lookup — seconds, not instant. The Panel echoes the question immediately, then shows an animated typing indicator (not a spinner: "composing a reply," not "system busy") with an optional grounded caption. The input disables to block double-submit. This pending bubble is the exact slot future streaming fills token-by-token.

What you're seeing, in spec terms (§5.3.1):

  • Optimistic echo — the question appears the instant it's sent, before an answer exists.
  • Animated typing indicator — dots over a spinner; reads as "replying."
  • Grounded caption — Checking the record… reassures the reply is FHIR-backed, not free-associated. Kept generic — no step we can't guarantee.
  • Input disabled — no double-submit; re-enables on success or error.
  • Streaming seam — the pending bubble is where tokens will later stream in. Building it now makes streaming a swap, not a rework.
